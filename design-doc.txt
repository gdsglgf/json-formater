Json 格式化工具

requirement
1. 传入一个json string
2. 格式化输出结果
	a. 如果格式不正确，输出错误信息
3. 把所有的记录存储到mysql
4. 客户端可以login
5. 客户端可以查询所有历史记录
6. 技术要求
	a. spring mvc 4.x.x 版本
	b. client 用 bootstrap
	c. orm 用 mybatis
	d. 出一份设计文档

转换功能
1. 用户输入一个json字符串。
2. 用户点击格式化按钮。
3. 如果json字符串格式是正确的，那么在屏幕上输出格式化之后的json字符串。
   如果格式不正确，输出错误信息。
4. 对已登录用户，将输入的合法的json字符串保存到后台数据库中，便于以后查询及使用。
   对未登录用户的数据不做处理。


查询功能
1. 用户登录系统。
2. 用户可以查询所有历史记录。
3. 显示历史记录。


登录功能
1. 用户输入用户名，密码。
2. 用户点击登录按钮。
3. 如果登录成功，进入用户主页。反之，提示用户用户名或密码不正确。


注册功能
1. 用户输入用户名，密码，邮箱。
2. 用户点击注册按钮。
3. 如果注册成功，进入用户主页。反之，提示用户名或邮箱已被注册。 


数据库设计

用户表
用户编号（自增），用户名（唯一），密码，邮箱（唯一），注册时间

历史信息表
信息编号（自增），用户编号（外键），历史信息（json字符串），提交时间
